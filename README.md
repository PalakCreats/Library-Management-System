# Library-Management-System
### 1. Project Overview
A simple **Library Management System** built using Python, Tkinter, and SQLite. It allows users to add and view book records through an easy-to-use graphical interface. The application stores data locally and ensures basic input validation. Ideal for beginners learning GUI development and database integration in Python.

### 2. Technologies Used
- Python – main programming language
- Tkinter – used to create the graphical user interface
- SQLite – used for local database storage

### 3. How the System Works
- The user enters book details such as title, author, and quantity
- The data is stored in an SQLite database
- Users can view all stored books using the view feature

### 4. Main Features
- Add new books
- Store data permanently in a database
- View all books in a table format
- Simple and easy-to-use interface
- Input validation to prevent empty fields

### 5. User Interface
The application has a clean and simple design built with Tkinter. It includes input fields for data entry, buttons for actions, and a table to display stored records.

### 6. Database Functionality
- Uses SQLite database (library.db)
- Automatically creates the table if it does not exist
- Stores and retrieves book data efficiently

### 7. Advantages
- Easy to use
- Lightweight and fast
- Works offline without internet
- Suitable for beginners learning Python

### 8. Future Improvements
- Add update and delete functionality
- Implement search feature
- Add user authentication system
- Integrate APIs for automatic book data fetching
